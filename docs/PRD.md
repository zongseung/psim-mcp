# PSIM-MCP Server: Product Requirements Document (PRD)

> **버전**: v1.0
> **작성일**: 2026-03-15
> **상태**: Draft

---

## 1. 제품 개요

### 1.1 제품명
**PSIM-MCP Server** — Claude Desktop에서 자연어로 Altair PSIM을 제어하는 MCP 서버

### 1.2 한 줄 요약
자연어 명령을 통해 PSIM 전력전자 시뮬레이션의 프로젝트 열기, 파라미터 변경, 시뮬레이션 실행, 결과 추출을 자동화하는 MCP(Model Context Protocol) 서버

### 1.3 배경 및 동기

**문제 정의**
- PSIM은 전력전자 분야의 핵심 시뮬레이션 도구이나, 모든 작업이 GUI 기반으로 이루어짐
- 반복적인 파라미터 스윕, 다수 회로의 시뮬레이션 실행, 결과 비교 등에 수작업이 과도하게 소요됨
- PSIM Python API가 존재하지만 활용도가 낮고, 자연어 기반 접근성이 없음

**기회**
- MCP를 통해 Claude Desktop과 PSIM을 연결하면, 자연어로 시뮬레이션 워크플로우를 자동화할 수 있음
- 전력전자 엔지니어가 "이 회로의 스위칭 주파수를 50kHz로 바꾸고 시뮬레이션 돌려줘" 같은 명령으로 작업 가능
- 외부 배포를 통해 PSIM 사용자 커뮤니티 전체에 가치를 제공할 수 있음

### 1.4 목표

| 구분 | 목표 |
|------|------|
| **단기** | Mac에서 MCP 서버 골격 완성, mock 기반 전체 워크플로우 검증 |
| **중기** | Windows에서 실제 PSIM 연동, Claude Desktop 통합 테스트 완료 |
| **장기** | 외부 배포 가능 수준의 안정성, 문서화, 패키징 달성 |

### 1.5 비목표 (Non-Goals)

- `.psimsch` 파일 포맷을 직접 리버스 엔지니어링하여 회로 생성기를 만드는 것
- PSIM GUI를 대체하는 것 (보조 도구로 위치)
- PSIM 라이선스 우회 또는 크랙
- 실시간 하드웨어-인-더-루프(HIL) 시뮬레이션 지원

## 2. 대상 사용자

### 2.1 주요 사용자 (Primary)
- **전력전자 엔지니어**: PSIM을 일상적으로 사용하며, 반복 작업 자동화를 원하는 사용자
- **연구자/대학원생**: 파라미터 스터디, 비교 시뮬레이션 등 대량 작업이 필요한 사용자

### 2.2 보조 사용자 (Secondary)
- **팀 리더/관리자**: 시뮬레이션 결과를 자연어로 요약받고 싶은 사용자
- **PSIM 입문자**: CLI보다 자연어가 편한 사용자

### 2.3 사용자 환경
- **OS**: Windows 10/11 (PSIM 실행 필수)
- **PSIM 버전**: Altair PSIM 2024/2025 (Python API 포함 버전)
- **Claude Desktop**: 최신 버전 (MCP 지원)
- **Python**: PSIM 번들 Python 3.8 + 시스템 Python 3.12+

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 핵심 기능 (P0 — Must Have)

#### FR-001: 프로젝트 열기 (`open_project`)
- **설명**: PSIM 프로젝트 파일(`.psimsch`)을 열고 현재 상태 정보를 반환
- **입력**: 프로젝트 파일 경로
- **출력**: 프로젝트 이름, 포함된 컴포넌트 목록, 현재 파라미터 값
- **검증**: 파일 존재 여부, 확장자 검증, 읽기 권한 확인

#### FR-002: 파라미터 설정 (`set_parameter`)
- **설명**: 열린 프로젝트의 컴포넌트 파라미터를 변경
- **입력**: 컴포넌트 이름/ID, 파라미터 이름, 새 값
- **출력**: 변경 전/후 값, 성공 여부
- **검증**: 컴포넌트 존재 여부, 파라미터 이름 유효성, 값 범위 검증

#### FR-003: 시뮬레이션 실행 (`run_simulation`)
- **설명**: 현재 열린 프로젝트의 시뮬레이션을 실행하고 결과를 반환
- **입력**: 시뮬레이션 옵션 (시간, 스텝 등 — 선택적)
- **출력**: 실행 상태, 소요 시간, 결과 파일 경로, 주요 결과 요약
- **검증**: 프로젝트가 열려있는지, 시뮬레이션 설정이 유효한지

#### FR-004: 결과 추출 (`export_results`)
- **설명**: 시뮬레이션 결과를 지정된 형식으로 내보내기
- **입력**: 출력 디렉터리, 형식 (JSON, CSV 등)
- **출력**: 내보낸 파일 경로, 데이터 요약
- **검증**: 출력 디렉터리 쓰기 권한, 결과 데이터 존재 여부

#### FR-005: 상태 조회 (`get_status`)
- **설명**: 현재 서버 및 PSIM 연결 상태를 반환
- **입력**: 없음
- **출력**: 모드(mock/real), PSIM 연결 상태, 열린 프로젝트 정보, 최근 시뮬레이션 결과

### 3.2 중요 기능 (P1 — Should Have)

#### FR-006: 파라미터 스윕 (`sweep_parameter`)
- **설명**: 하나의 파라미터를 범위 내에서 변경하며 반복 시뮬레이션
- **입력**: 파라미터, 시작값, 끝값, 스텝
- **출력**: 각 값별 시뮬레이션 결과 요약, 비교 테이블

#### FR-007: 프로젝트 정보 조회 (`get_project_info`)
- **설명**: 열린 프로젝트의 상세 구조 정보 반환
- **입력**: 없음 (현재 열린 프로젝트 기준)
- **출력**: 컴포넌트 트리, 파라미터 목록, 연결 정보

#### FR-008: 결과 비교 (`compare_results`)
- **설명**: 두 시뮬레이션 결과를 비교하여 차이점 요약
- **입력**: 두 결과 파일 경로
- **출력**: 차이 요약, 주요 지표 비교 테이블

### 3.3 부가 기능 (P2 — Nice to Have)

#### FR-009: 회로 요소 추가 (`add_component`)
- Python API 기반으로 새 컴포넌트를 회로에 추가

#### FR-010: 넷리스트 생성/분석 (`generate_netlist`)
- XML 넷리스트를 생성하고 구조를 분석하여 반환

#### FR-011: 템플릿 관리 (`list_templates` / `load_template`)
- 미리 준비된 회로 템플릿 목록 조회 및 로드

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 성능
| 항목 | 요구사항 |
|------|----------|
| tool 응답 시간 (mock) | < 500ms |
| tool 응답 시간 (real, 시뮬레이션 제외) | < 3s |
| 시뮬레이션 타임아웃 | 설정 가능, 기본 300s |
| 동시 요청 | 순차 처리 (PSIM은 단일 인스턴스) |

### 4.2 안정성
- 예외 발생 시 서버가 크래시하지 않고 에러 메시지를 반환
- PSIM 프로세스 비정상 종료 시 감지 및 복구
- 모든 tool 호출에 대한 구조화된 로깅

### 4.3 보안
- 파일 시스템 접근을 허용된 디렉터리로 제한 (path traversal 방지)
- `.env` 파일에 민감 정보를 저장하고 git에서 제외
- 원격 배포 시 인증/인가 레이어 필요

### 4.4 이식성
- Mac/Linux: mock 모드로 개발 및 테스트 가능
- Windows: real 모드로 PSIM 연동
- 환경 변수로 모든 경로와 모드를 제어

### 4.5 배포
- PyPI 패키지로 배포 가능한 구조 (`pip install psim-mcp`)
- Claude Desktop 설정 파일 예시 제공
- Docker 지원은 향후 검토 (PSIM이 Windows GUI 앱이므로 제약 있음)

## 5. 사용자 시나리오 (User Stories)

### US-001: 기본 시뮬레이션 워크플로우
> "Buck 컨버터 프로젝트를 열고, 스위칭 주파수를 100kHz로 변경한 다음 시뮬레이션을 돌려서 출력 전압 파형을 보여줘"

**수락 기준**:
1. 프로젝트 파일을 정상적으로 열 수 있다
2. 파라미터를 변경하고 변경 전/후 값을 확인할 수 있다
3. 시뮬레이션을 실행하고 결과를 텍스트로 요약받을 수 있다
4. 결과 파일이 지정된 경로에 저장된다

### US-002: 파라미터 스터디
> "인덕턴스를 10uH에서 100uH까지 10uH 단위로 바꿔가면서 각각 시뮬레이션 돌리고 출력 리플 비교 테이블 만들어줘"

**수락 기준**:
1. 파라미터 범위를 지정하여 반복 시뮬레이션을 실행할 수 있다
2. 각 결과를 비교하는 요약 테이블을 반환한다

### US-003: 상태 확인 및 디버깅
> "지금 PSIM 연결 상태 어때? 마지막 시뮬레이션 결과도 다시 보여줘"

**수락 기준**:
1. 서버 모드, PSIM 연결 상태, 열린 프로젝트 정보를 반환한다
2. 최근 시뮬레이션 결과에 접근할 수 있다

### US-004: 에러 상황 대응
> "이 경로에 있는 프로젝트 열어줘" (존재하지 않는 경로)

**수락 기준**:
1. 명확한 한국어 에러 메시지를 반환한다
2. 서버가 크래시하지 않는다
3. 에러 내용이 로그에 기록된다

## 6. 기술 제약사항 (Constraints)

### 6.1 PSIM 의존성
| 항목 | 제약 |
|------|------|
| 실행 환경 | Windows만 지원 (PSIM이 Windows 전용) |
| Python 버전 | PSIM 번들 Python 3.8 (API 실행용) |
| 라이선스 | 유효한 PSIM 라이선스 필요 |
| 동시성 | PSIM 단일 인스턴스만 실행 가능 |

### 6.2 MCP 프로토콜 제약
| 항목 | 제약 |
|------|------|
| SDK | `mcp` Python 패키지 (v1.x) |
| 전송 방식 | stdio (Claude Desktop 로컬), streamable-http (원격) |
| Python 버전 | 3.10+ (MCP SDK 요구사항) |

### 6.3 이중 Python 환경
- **MCP 서버**: Python 3.12 (시스템 Python)
- **PSIM API 호출**: Python 3.8 (PSIM 번들)
- 해결 방안: MCP 서버에서 subprocess로 PSIM 번들 Python을 호출하거나, 호환 가능한 래퍼 사용

## 8. 리스크 및 완화 전략

| 리스크 | 영향도 | 완화 전략 |
|--------|--------|-----------|
| PSIM Python API 문서 부족 | 높음 | Save as Python Code로 API 패턴 학습, 커뮤니티/Altair 지원 활용 |
| PSIM 버전별 API 차이 | 중간 | adapter 패턴으로 버전별 분기 가능하게 설계 |
| 이중 Python 환경 충돌 | 중간 | subprocess 기반 호출로 환경 격리 |
| Windows 경로/권한 문제 | 높음 | 경로 검증 유틸리티 선제 구현, 상세 로깅 |
| PSIM 라이선스 제약 | 중간 | mock 모드로 라이선스 없이 개발/테스트 가능하게 유지 |
| Altair-Siemens 인수 후 API 변경 | 낮음 | adapter 패턴으로 교체 가능하게 설계 |

## 10. 용어 정의

| 용어 | 정의 |
|------|------|
| **PSIM** | Altair의 전력전자 시뮬레이션 소프트웨어 |
| **MCP** | Model Context Protocol — AI 모델이 외부 도구와 상호작용하는 프로토콜 |
| **FastMCP** | MCP Python SDK의 고수준 서버 프레임워크 |
| **Tool** | MCP에서 AI 모델이 호출할 수 있는 함수 단위 |
| **Adapter** | PSIM 실행 환경을 추상화하는 인터페이스 구현체 |
| **Mock 모드** | PSIM 없이 더미 데이터로 동작하는 개발/테스트 모드 |
| **Real 모드** | 실제 PSIM과 연동하여 동작하는 운영 모드 |
| **`.psimsch`** | PSIM 회로 스키매틱 파일 형식 |
| **`.smv`** | PSIM SIMVIEW 결과 파일 형식 |
